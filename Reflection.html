The concept of resposive design is crucial part in web development. Teams that work on this make sure that the website created is
accesible to every one throught every device (phone,tablet,desktop) but main challenge is to create the design such that the
content is the same across devices. RWD is a meathod which changes the appearence of the webpage based on screen size and orientation
The design uses breakpoint to find how the layout of the site should be visible. One design will be above the breakpoint and other will
be below the point and break point determines the height width of the device for the desin to be applied. The same HTML is served to all devices
using CSS to change the appearence of the page. Instead of creating differetn HTML and CSS for different devices we can use 1 codebase for all
devices. In RWD the content will be same but the layout grows or shrinks according the size. A three column desktop design will change to 2 column
on a tablet to a 1 column in smartphone. Also, navigation bars can be seen in the desktop but in smart phone it will be hidden in 3 lines to open
the navigation bar.

Because responsive design relies on shuffling elements around the page, design and development need to work closely together to ensure a usable
experience across devices. Responsive design often turns into solving a puzzle. However, ensuring that elements fit within a page is not enough.
For a responsive design to be sucessfull, the design must also be usable at all screen resolutions and sizes. When elements move around the page
the user experince can be completely different from one view of the site to the next. It is important that design and development tems work together
not to just determine how the content should be shuffled around, but to also see what the end of that shift looks like and how it affects the user
experience. Many teams look for popular responsive design frameworks, such as Bootstraps to help create designs.Such frameworks can be great help in
moving development along. However, carefully consider how the framework will work with the content and functionality of the site, rather than how it works
in general.
